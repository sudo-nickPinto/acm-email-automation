# Lesson 07 — Setup Wizard and .env

## What setup_wizard.py does

The setup wizard is a guided, 5-step interactive questionnaire that collects the user's preferences and writes them to a `.env` file. It is designed for non-technical users — every prompt explains what it is asking and why.

## stdin handling

```python
if not sys.stdin.isatty():
    sys.stdin = open("/dev/tty", "r")
```

This is the Python equivalent of `start.sh`'s `exec < /dev/tty`. When invoked from `curl | bash`, Python's `sys.stdin` is the pipe (already exhausted), so `input()` would raise `EOFError`. Reopening from `/dev/tty` reconnects to the real terminal.

## Terminal formatting

The wizard uses ANSI escape codes for colored, styled output:

```python
BOLD = "\033[1m"    # Bold text
DIM  = "\033[2m"    # Dimmed/gray text (for hints)
GREEN = "\033[0;32m" # Success messages
YELLOW = "\033[1;33m" # Warnings
RED  = "\033[0;31m"  # Error messages
CYAN = "\033[0;36m"  # Banners and dividers
NC   = "\033[0m"     # "No Color" — resets formatting
```

**How ANSI codes work:** These are special character sequences that terminals interpret as formatting instructions. `\033[` is the "escape" prefix, followed by a number that selects the style, and `m` terminates the sequence. `\033[0m` resets all formatting.

The helper functions (`banner()`, `info()`, `success()`, `warn()`, `fail()`, `step()`) wrap these codes so the rest of the code reads cleanly: `success("Email saved")` instead of `print(f"  \033[0;32m✔\033[0m  Email saved")`.

## Step 1: Source selection (_prompt_sources)

```python
def _prompt_sources() -> list[NewsSource]:
    for i, source in enumerate(AVAILABLE_SOURCES, start=1):
        print(f"    {BOLD}[{i}]{NC} {source.name}")
        print(f"        {DIM}{source.description}{NC}")
```

The function:
1. Displays all available sources with numbered labels
2. Accepts space-separated numbers (e.g., `1 3 4`) or `all`
3. Validates that numbers are in range
4. Deduplicates while preserving order (using a set + list pattern)
5. Returns a list of `NewsSource` objects

**Why deduplicate?** If a user types `1 1 3`, we do not want to fetch ACM TechNews twice. The `seen` set tracks which numbers have been processed.

## Step 2: Email (_prompt_email)

```python
email_regex = re.compile(r"^[a-zA-Z0-9._%+-]+@gmail\.com$")
```

The regex validates that the input looks like a Gmail address. It is not exhaustive email validation (RFC 5322 is extremely complex), but it catches common mistakes:
- Missing `@` symbol
- Typos like `@gmal.com`
- Non-Gmail addresses (the tool only supports Gmail SMTP)

## Step 3: App Password (_prompt_app_password)

This is the most complex step because it teaches users a concept they have likely never encountered:

```python
password = getpass.getpass(f"  Your App Password: ").strip()
password = password.replace(" ", "")  # Google shows it with spaces
```

**Why `getpass`?** It hides the user's typing so the password does not appear on screen. This is standard security practice for password entry.

**Why replace spaces?** Google displays App Passwords as `abcd efgh ijkl mnop` (with spaces for readability), but the actual password is `abcdefghijklmnop` (16 contiguous letters).

**Validation rules:**
- Exactly 16 characters (after removing spaces)
- Only letters (no numbers or symbols)
- Not empty

## Step 4: Schedule (_prompt_schedule)

```python
def _prompt_schedule() -> str:
    current_os = detect_os()
    if current_os == "macos":
        mechanism = "macOS LaunchAgent"
    elif current_os == "linux":
        mechanism = "cron job"
    elif current_os == "windows":
        mechanism = "Windows Task Scheduler"
    else:
        # Unsupported OS — skip scheduling
        return ""
```

The function:
1. Detects the OS and names the scheduling mechanism
2. Asks if the user wants automatic delivery
3. Collects the desired time in 24-hour format (validates HH:MM)
4. Calls `install_schedule(hour, minute)` from `scheduler.py`
5. Returns the time string (e.g., `"08:00"`) or empty string if declined

## Writing the .env file

```python
def _write_env(email, app_password, sources, schedule_time=""):
    source_keys = ",".join(s.key for s in sources)
    env_content = (
        "# News Digest Configuration\n"
        "# Generated by setup wizard\n"
        f"SMTP_EMAIL={email}\n"
        f"SMTP_APP_PASSWORD={app_password}\n"
        f"RECIPIENT_EMAIL={email}\n"
        f"SELECTED_SOURCES={source_keys}\n"
        f"SCHEDULE_TIME={schedule_time}\n"
    )
    ENV_FILE.write_text(env_content)
    _protect_secret_file(ENV_FILE)
```

**Why does RECIPIENT_EMAIL equal SMTP_EMAIL?** Most users send the digest to themselves. The field exists as a separate config value so that a power user could send their digest to a different address by manually editing `.env`, but the wizard defaults to self-delivery.

**What does _protect_secret_file do?**

```python
def _protect_secret_file(path: Path) -> None:
    if os.name == "nt":
        return  # Windows has a different permission model
    try:
        path.chmod(0o600)
    except OSError:
        warn(f"Couldn't tighten permissions for {path.name}.")
```

`chmod 600` means "only the owner can read or write this file." On a shared computer, this prevents other users from reading your Gmail App Password. On Windows, the NTFS permission model is different, so this step is skipped (Windows file ACLs are not set from Python in this project).

## Step 5: Test email (_offer_test)

```python
result = subprocess.run(
    [str(_venv_python()), str(PROJECT_ROOT / "main.py")],
    cwd=str(PROJECT_ROOT),
    capture_output=True, text=True,
)
```

The test runs `main.py` as a subprocess rather than importing and calling `main()` directly. **Why?** Because `config.py` was already imported when the wizard started, and it loaded the *old* `.env` (which did not exist yet). Running a fresh Python process ensures `config.py` reads the newly-written `.env` file.

## Reconfiguration flow

If `.env` already exists when the wizard starts:

```python
if ENV_FILE.exists():
    warn("You already have a configuration file (.env).")
    answer = input("  Do you want to reconfigure? [y/n]: ")
    if answer not in ("y", "yes"):
        success("Keeping existing configuration. You're all set!")
        return
```

This prevents accidentally overwriting a working configuration. The user must explicitly confirm they want to reconfigure.
