# Lesson 12 — CLI Management Layer

> **Goal:** Understand how `newsdigest/cli.py` provides a full
> interactive menu for managing the digest after installation.

---

## 1  Purpose

The CLI is the **post-install management interface**.  Once `news-digest`
is installed, typing `news-digest` (with no arguments) launches this
interactive menu.  It gives non-technical users a way to:

- Send or preview the digest
- Change settings (email, password, sources, schedule)
- View status
- Uninstall completely

---

## 2  Architecture

```
news-digest (bash/cmd wrapper)
  └─ python3 -m newsdigest.cli   (or direct import)
       └─ show_menu()
            └─ while True loop
                 ├─ Display menu
                 ├─ Read choice
                 └─ Dispatch to action_*() function
```

### 2.1  Menu Options Table

```python
MENU_OPTIONS = [
    ("1", "Send digest now",           action_send_now),
    ("2", "Preview digest (dry run)",  action_preview),
    ("3", "Force re-send digest",      action_send_force),
    ("4", "Change news sources",       action_change_sources),
    ("5", "Change email address",      action_change_email),
    ("6", "Change app password",       action_change_password),
    ("7", "Change delivery schedule",  action_change_schedule),
    ("8", "View status",              action_status),
    ("9", "Uninstall",                action_uninstall),
    ("0", "Exit",                     None),
]
```

Each option is a `(key, label, function)` tuple.  The dispatch loop
simply matches the user's input against keys and calls the function.

## 3  Terminal Formatting

```python
BOLD = "\033[1m"
DIM = "\033[2m"
GREEN = "\033[0;32m"
YELLOW = "\033[1;33m"
RED = "\033[0;31m"
CYAN = "\033[0;36m"
BLUE = "\033[0;34m"
WHITE = "\033[1;37m"
NC = "\033[0m"    # No Color (reset)
```

ANSI escape codes provide colors and styling without any external
library.  Every styled string ends with `{NC}` to reset formatting.

Key formatting helpers:
- `_clear()` — clears the screen (only when stdout is a real terminal)
- `_header()` — main menu title with decorative borders
- `_banner()` — sub-screen titles
- `_success()`, `_warn()`, `_fail()`, `_info()` — status messages with icons

## 4  .env File Management

### 4.1  Reading

```python
def _read_env() -> dict[str, str]:
    env = {}
    if ENV_FILE.exists():
        for line in ENV_FILE.read_text().splitlines():
            line = line.strip()
            if line and not line.startswith("#") and "=" in line:
                key, _, value = line.partition("=")
                env[key.strip()] = value.strip()
    return env
```

**Why not use python-dotenv?**  The CLI needs to **write** the `.env`
file too.  `python-dotenv` is read-only for our purposes.  A simple
key=value parser handles both reading and writing consistently.

### 4.2  Writing

```python
def _write_env(env: dict[str, str]) -> None:
    lines = ["# News Digest Configuration",
             f"# Generated by setup wizard -- re-run {_setup_command()} to change"]
    for key in ("SMTP_EMAIL", "SMTP_APP_PASSWORD", "RECIPIENT_EMAIL",
                "SELECTED_SOURCES", "SCHEDULE_TIME"):
        if key in env:
            lines.append(f"{key}={env[key]}")
    ENV_FILE.write_text("\n".join(lines) + "\n")
    _protect_env_file()
```

Keys are written in a **fixed order** to keep the file tidy.
`_protect_env_file()` runs `chmod 600` on POSIX systems to prevent
other users from reading credentials.

### 4.3  Reloading Config

```python
def _reload_config():
    import newsdigest.config
    for key in ("SMTP_EMAIL", "SMTP_APP_PASSWORD", "RECIPIENT_EMAIL",
                "SELECTED_SOURCES", "SCHEDULE_TIME"):
        os.environ.pop(key, None)
    importlib.reload(newsdigest.config)
    return newsdigest.config
```

After changing `.env`, the in-memory config is stale.  This function:
1. Clears the old env vars from `os.environ`
2. Uses `importlib.reload()` to re-execute `config.py`
3. `config.py` calls `load_dotenv()` again, reading the updated file

## 5  Action Functions

### 5.1  Send Actions

`action_send_now()`, `action_preview()`, and `action_send_force()` all
delegate to `main.py` via `subprocess.run()`.  They invoke the venv
Python directly:

```python
subprocess.run(
    [str(VENV_PYTHON), str(PROJECT_ROOT / "main.py"), "--dry-run"],
    cwd=str(PROJECT_ROOT),
)
```

**Why subprocess instead of importing main?**  Because `main.py` calls
`sys.exit()` on certain conditions, which would terminate the CLI too.
Running as a subprocess isolates the exit behavior.

### 5.2  Change Sources

`action_change_sources()` displays all available sources with checkmarks
for currently selected ones, accepts a space-separated list of numbers
(or "all"), validates input, updates `.env`, and confirms.

Deduplication of user input: `seen = set()` prevents the same source
from being selected twice.

### 5.3  Change Email

Validates with a regex: `r"^[a-zA-Z0-9._%+-]+@gmail\.com$"`.
**Gmail-only** — the project only supports Gmail SMTP.

### 5.4  Change Password

Uses `getpass.getpass()` for hidden input.  Validates:
- Exactly 16 characters
- Letters only (no numbers or symbols)
- Strips spaces (app passwords are often displayed with spaces)

### 5.5  Change Schedule

Integrates with `newsdigest.scheduler`:
- Shows current schedule status
- Options: set a new time, or turn off scheduling
- Validates HH:MM format with a regex
- Calls `install_schedule()` or `uninstall_schedule()`

### 5.6  Status

Displays a dashboard:
- Install path
- Python version
- Email address
- Selected sources (resolved from keys to names)
- Schedule status
- Last sent timestamp (from `.last_sent` file mtime)

### 5.7  Uninstall

**Destructive action** — requires typing "yes" to confirm.

Steps:
1. Remove the daily schedule (if active)
2. Detect symlink location (`/usr/local/bin` or `~/.local/bin`, or
   Windows `AppData`)
3. `shutil.rmtree()` the project directory
4. Remove the symlink/wrapper
5. `sys.exit(0)` — nothing left to do

## 6  The Menu Loop

```python
def show_menu() -> None:
    if not sys.stdin.isatty():
        try:
            sys.stdin = open("/dev/tty", "r")
        except OSError:
            pass

    while True:
        _clear()
        _header("News Digest")
        # ... display menu ...
        choice = input("  ▸ ").strip()
        if choice == "0":
            break
        # dispatch to action
```

### 6.1  stdin Reconnection

```python
if not sys.stdin.isatty():
    sys.stdin = open("/dev/tty", "r")
```

**Why?**  When the CLI is launched via `curl | bash` or a pipe, stdin is
the pipe, not the keyboard.  Reopening `/dev/tty` reconnects to the
actual terminal so `input()` works.

### 6.2  Screen Clear

`_clear()` uses ANSI escape `\033[2J\033[H` — clears the screen and
moves the cursor to the top-left.  Only runs when stdout is a real
terminal (not redirected to a file).

## 7  Key Takeaways

| Concept | Implementation |
|---------|---------------|
| Menu-driven UX | `while True` loop with numeric dispatch |
| ANSI formatting | Hardcoded escape codes, no external library |
| .env manipulation | Custom read/write for full control |
| Config hot-reload | `os.environ.pop()` + `importlib.reload()` |
| Process isolation | `subprocess.run()` for main.py calls |
| Security | `chmod 600` on .env, `getpass` for passwords |
