# 04 — Config and Environment

## The Problem Config Solves

Imagine if `emailer.py` had your Gmail password hard-coded:

```python
server.login("you@gmail.com", "abcdefghijklmnop")
```

That would work. But it's dangerous and inflexible:

1. **Security** — the password is visible to anyone who reads the code (or the git history)
2. **Portability** — someone else using this project would have to edit the Python code
3. **Separation of concerns** — the emailer's job is to send email, not to know where credentials come from

`config.py` solves all three by acting as a translation layer between the `.env` file and the rest of the codebase.

## File Walkthrough: `config.py`

### Imports and dotenv

```python
import os
from dotenv import load_dotenv

load_dotenv()
```

`load_dotenv()` reads the `.env` file and injects each `KEY=VALUE` pair into the process environment. This **must** be called before any `os.getenv()` calls below.

### SMTP Settings

```python
SMTP_EMAIL: str | None = os.getenv("SMTP_EMAIL")
SMTP_APP_PASSWORD: str | None = os.getenv("SMTP_APP_PASSWORD")
```

The Gmail login credentials. `os.getenv("KEY")` returns the value of the environment variable, or `None` if it's not set.

```python
SMTP_SERVER: str = "smtp.gmail.com"
SMTP_PORT: int = 587
```

These are hard-coded because they never change — every Gmail user connects to the same server on the same port.

- **Port 587** — SMTP with STARTTLS (starts unencrypted, upgrades to TLS)
- **Port 465** — older "implicit TLS" (encrypted from the start). Also works, but 587 is the modern standard
- **Port 25** — unencrypted SMTP. Never use this for authenticated mail

### Recipient

```python
RECIPIENT_EMAIL: str | None = os.getenv("RECIPIENT_EMAIL", os.getenv("SMTP_EMAIL"))
```

Defaults to the sender's own address. Most users email themselves the digest, but they can override this in `.env` to send to a different address.

### Source Selection

```python
SELECTED_SOURCES_RAW: str = os.getenv("SELECTED_SOURCES", "")

SELECTED_SOURCES: list[str] = [
    s.strip() for s in SELECTED_SOURCES_RAW.split(",") if s.strip()
]
```

This is new in v2. The setup wizard writes a comma-separated list of source keys to `.env` (e.g., `SELECTED_SOURCES=acm_technews,bbc_tech`). Config parses this into a clean Python list: `["acm_technews", "bbc_tech"]`.

The list comprehension handles edge cases — extra whitespace, trailing commas, empty strings.

## The `.env` File

Created automatically by the setup wizard. Example:

```env
# News Digest Configuration
# Generated by setup wizard — re-run ./start.sh to change
SMTP_EMAIL=yourname@gmail.com
SMTP_APP_PASSWORD=abcdefghijklmnop
RECIPIENT_EMAIL=yourname@gmail.com
SELECTED_SOURCES=acm_technews,mit_tech_review,bbc_tech
```

**Key rules:**
- No quotes around values
- No spaces around `=`
- Lines starting with `#` are comments
- This file is **never committed to git** (listed in `.gitignore`)

## The `.env.example` File

```env
# Gmail SMTP credentials
SMTP_EMAIL=your_gmail@gmail.com
SMTP_APP_PASSWORD=your_app_password_here

# Recipient (defaults to sender if omitted)
RECIPIENT_EMAIL=your_gmail@gmail.com

# Sources (comma-separated keys — see newsdigest/sources.py for available keys)
SELECTED_SOURCES=acm_technews,mit_tech_review,nytimes_tech,bbc_tech
```

This template **is** committed to git so anyone cloning the repo knows what variables are needed. But in practice, the setup wizard handles everything — users never need to touch this file.

## Gmail App Password Setup

Gmail requires a special "App Password" when 2-Factor Authentication is enabled. The setup wizard explains all of this interactively, but here's the reference:

1. Go to [Google Account → Security](https://myaccount.google.com/security)
2. Make sure 2-Step Verification is enabled
3. Go to [App Passwords](https://myaccount.google.com/apppasswords)
4. Enter "News Digest" as the app name
5. Click "Create"
6. Copy the 16-character password

The password looks like `abcd efgh ijkl mnop` (spaces are cosmetic — Gmail ignores them). The setup wizard strips spaces automatically.

## How Config Gets Used

```python
# In emailer.py:
from newsdigest.config import RECIPIENT_EMAIL, SMTP_APP_PASSWORD, SMTP_EMAIL, SMTP_PORT, SMTP_SERVER

# In scraper.py:
from newsdigest.config import SELECTED_SOURCES
```

Both modules import what they need by name. They don't know about `.env` files or `os.getenv` — `config.py` has already resolved everything into simple constants.

If you ever want to change how configuration is loaded (say, from a different file format or a cloud service), you only change `config.py`. The scraper and emailer don't need to be touched.

## Common Mistakes

| Mistake | What happens | Fix |
|---------|-------------|-----|
| Forgot to run the wizard | `SMTP_EMAIL` is `None`, login fails | Run `./start.sh` |
| Used regular Gmail password | Gmail rejects: "Application-specific password required" | Generate an App Password |
| Committed `.env` to git | Secrets exposed in git history | Add to `.gitignore`, rotate credentials |
| No sources selected | `SELECTED_SOURCES` is empty, script exits | Re-run `./start.sh` and pick sources |
